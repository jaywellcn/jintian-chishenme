# 「今天吃什么」小程序 — 上线操作保姆级指南

> 本指南覆盖第③步到第⑥步，面向零技术背景用户。
> 
> 前置条件：你已完成 ①注册小程序账号（拿到AppID）和 ②下载微信开发者工具。

---

## 第③步：导入项目 + 填入 AppID

### 3.1 打开微信开发者工具

1. 双击桌面上的「微信开发者工具」图标
2. 用你注册小程序的微信号扫码登录

### 3.2 导入项目

1. 在左侧菜单点击「**小程序**」标签页
2. 点击右上角「**+**」号（或「导入项目」按钮）
3. 在弹窗中填写：
   - **项目目录**：点击「目录」按钮，选择你电脑上的 `今天吃什么-miniapp` 文件夹
   - **AppID**：填入你在第①步注册时拿到的 AppID（格式类似 `wx1a2b3c4d5e6f7g8h`）
   - **项目名称**：自动填充为「今天吃什么」，不用改
4. 点击「**确定**」

> ⚠️ 如果提示「未找到 project.config.json」，说明你选错了目录——要选 `今天吃什么-miniapp` 这个文件夹本身，不要选它的上层或下层。

### 3.3 修改 project.config.json 中的 AppID

1. 在开发者工具左侧的文件树中，找到根目录的 `project.config.json` 文件
2. 双击打开它
3. 找到第 33 行：
   ```
   "appid": "YOUR_APPID_HERE",
   ```
4. 把 `YOUR_APPID_HERE` 替换成你的真实 AppID，例如：
   ```
   "appid": "wx1a2b3c4d5e6f7g8h",
   ```
5. 按 `Ctrl + S`（Mac 按 `Cmd + S`）保存

> 💡 通常导入项目时填了 AppID，工具会自动写入。但建议打开确认一下是否正确。

### 3.4 此时先别急着编译

导入后你可能会看到报错，这是正常的——因为还没开通云开发。继续往下做第④步。

---

## 第④步：开通云开发 + 填入环境ID

### 4.1 开通云开发

1. 在开发者工具顶部工具栏，找到「**云开发**」按钮（一个云朵图标，通常在右上角）
2. 点击它，会弹出一个开通引导窗口
3. 点击「**开通**」
4. 输入一个环境名称，比如填 `prod` 或 `jintianchishenme`
5. **套餐选择**：选择「**按量付费**」（不用担心，开发阶段几乎不花钱，有免费额度）
6. 点击「**确定**」

> ⚠️ 开通时会要求你进行实名认证，按提示操作即可（需要微信绑定的身份证）。

### 4.2 获取环境ID

1. 开通完成后，会自动打开「云开发控制台」窗口
2. 在左侧菜单点击「**设置**」
3. 在「环境ID」一栏，你会看到一串类似 `prod-1a2b3c` 或 `jintianchishenme-4d5e6f` 的文字
4. **复制这串文字**（这就是你的环境ID）

### 4.3 填入 app.js

1. 回到开发者工具的代码编辑区
2. 在左侧文件树中，依次展开 `miniprogram/` → 找到 `app.js`，双击打开
3. 找到第 9 行：
   ```
   env: 'prod-xxxxxxxxx', // 请替换为你的云开发环境ID
   ```
4. 把 `prod-xxxxxxxxx` 替换成你刚才复制的环境ID，例如：
   ```
   env: 'jintianchishenme-4d5e6f',
   ```
5. 按 `Ctrl + S` 保存

### 4.4 验证云开发是否生效

1. 点击工具栏的「**编译**」按钮（一个播放按钮 ▶）
2. 如果没有报错，说明云开发初始化成功
3. 模拟器中可能会显示加载中或报错（因为数据库还没有数据），这是正常的

## 第⑤步：创建数据库集合 + 导入菜谱数据

### 5.1 打开云开发控制台

1. 点击开发者工具顶部的「**云开发**」按钮
2. 在弹出的云开发控制台窗口中，点击左侧菜单的「**数据库**」

### 5.2 创建 recipes 集合（菜谱数据表）

1. 在数据库页面，点击「**+**」号（或「添加集合」按钮）
2. 输入集合名称：`recipes`（必须全小写，完全一致）
3. 点击「确定」

### 5.3 创建 users 集合（用户数据表）

1. 再次点击「**+**」号
2. 输入集合名称：`users`（必须全小写，完全一致）
3. 点击「确定」

> 现在你的数据库里应该有两个集合：`recipes` 和 `users`

### 5.4 准备导入数据

菜谱数据文件已经在项目中了，路径是：
```
miniprogram/data/seed_recipes.json
```

这个文件包含 155 道菜谱。但云开发控制台导入需要 **JSON Lines 格式**（每行一个JSON对象，不是一个大数组）。我们需要先转换一下格式。

#### 转换方法（二选一）：

**方法A：用在线工具转换（推荐小白）**

1. 打开浏览器，访问任意 JSON 格式转换工具
2. 我帮你直接生成转换好的文件了，见下方

**方法B：我已经帮你生成好了**

转换后的文件已经放在：
```
miniprogram/data/seed_recipes_import.json
```

> 如果这个文件不存在，请告诉我，我立刻生成。

### 5.5 导入菜谱数据

1. 在云开发控制台 → 数据库 → 点击 `recipes` 集合
2. 点击右侧的「**导入**」按钮
3. 在导入弹窗中：
   - **选择文件**：点击「选择文件」，找到项目目录下的 `miniprogram/data/seed_recipes_import.json`
   - **导入格式**：选择「**JSON Lines**」（重要！不是 JSON）
   - **冲突处理**：选择「插入」（默认即可）
4. 点击「**确定**」开始导入
5. 等待导入完成，会提示「成功导入 155 条记录」

### 5.6 验证数据

1. 在 `recipes` 集合中，点击「记录总数」
2. 应该显示 **155 条**
3. 随便点开一条记录，确认有 `name`、`solar_term`、`main_ingredients` 等字段

### 5.7 设置数据库权限

这一步很重要，否则小程序无法读取数据：

1. 在 `recipes` 集合中，点击右上角的「**权限设置**」
2. 将权限改为「**所有用户可读，仅创建者可读写**」
3. 点击「保存」
4. 对 `users` 集合做同样的操作：权限改为「**仅创建者可读写**」

## 第⑥步：上传并部署云函数

项目里有 4 个云函数，需要逐个上传部署：

```
cloudfunctions/
├── getRecommendation/    ← 推荐引擎（核心）
├── getRecipes/           ← 菜谱查询
├── getUserProfile/       ← 用户偏好获取
└── updatePreferences/   ← 用户偏好更新
```

### 6.1 上传部署 getRecommendation（推荐引擎）

1. 在开发者工具左侧文件树中，找到 `cloudfunctions/` 目录
2. 展开它，找到 `getRecommendation` 文件夹
3. **右键**点击 `getRecommendation` 文件夹
4. 在右键菜单中选择「**上传并部署：云端安装依赖**」（注意：不是「不上传node_modules」）
5. 等待上传完成，底部状态栏会显示「上传成功」
6. 如果弹出提示「是否安装依赖」，选择「是」

> ⏳ 这个过程可能需要 30-60 秒，因为云端要安装 `wx-server-sdk` 依赖包。耐心等待底部提示「上传云函数成功」。

### 6.2 上传部署 getRecipes

1. 右键点击 `getRecipes` 文件夹
2. 选择「**上传并部署：云端安装依赖**」
3. 等待上传成功

### 6.3 上传部署 getUserProfile

1. 右键点击 `getUserProfile` 文件夹
2. 选择「**上传并部署：云端安装依赖**」
3. 等待上传成功

### 6.4 上传部署 updatePreferences

1. 右键点击 `updatePreferences` 文件夹
2. 选择「**上传并部署：云端安装依赖**」
3. 等待上传成功

### 6.5 验证云函数部署成功

1. 点击开发者工具顶部的「**云开发**」按钮
2. 在云开发控制台中，点击左侧「**云函数**」
3. 你应该能看到 4 个云函数：
   - `getRecommendation`
   - `getRecipes`
   - `getUserProfile`
   - `updatePreferences`
4. 每个函数的状态应该都是「正常」
5. 如果某个函数显示「异常」，右键该函数 → 「查看日志」，看具体报错

### 首次编译运行

1. 回到开发者工具主界面
2. 点击工具栏的「**编译**」按钮（▶）
3. 在左侧模拟器中，你应该看到：

   - 按引导完成城市选择、口味设置等

### 如果遇到问题

| 问题 | 解决方案 |
|------|----------|
| 白屏 | 检查 app.js 里的环境ID是否正确 |
| 提示「云函数不存在」 | 确认4个云函数都上传部署成功了 |
| 提示「集合不存在」 | 确认 recipes 和 users 集合已创建 |
| 推荐页空白 | 检查 recipes 集合是否有数据（应有155条） |
| 收藏功能报错 | 检查 users 集合权限设置是否正确 |
| 定位失败 | 正常现象，会默认使用北京，不影响使用 |
